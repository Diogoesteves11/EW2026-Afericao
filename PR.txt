 A persistência de dados é assegurada por uma base de dados MongoDB, integrada numa infraestrutura contentorizada via Docker Compose que orquestra três serviços: a base de dados (mongodb), uma API de dados (dataAPI na porta 16025) e uma interface web (webInterface na porta 16026). O processo de ingestão de dados envolveu um script Python (prepair.py) para converter o dataset original num formato compatível com o mongoimport, sendo a importação automatizada através de um script de inicialização shell no container de base de dados. A API de dados, construída com Express e Mongoose, disponibiliza endpoints para operações CRUD e filtragem por ano e marca. Para garantir a integridade do routing na interface web (Pug).
